# completion summary

This handoff artifact is optional by default. The required completion signal for kanban-backed runner work is a lead-visible `done` state on the active root kickoff card plus persisted delivery artifacts/state that the lead can inspect during release review. Use a completion summary when an explicit runner-to-lead wrap-up message is helpful or when a project overlay requires it.

## Minimum payload

- scope delivered
- artifacts produced
- verification executed
- unresolved risks
- suggested next step
